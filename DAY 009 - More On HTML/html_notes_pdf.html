HTML Notes

HTML (HyperText Markup Language)

HTML is basically the base of every website you see. It tells the browser how to structure and display the content. Think of it like the skeleton of the webpage.

  • Hyper text: the text which can be linked to other texts or resources.
  • Markup: It's called a markup language because it uses tags (like <p>, <h1>, <img>) to "mark up" text and other content, telling the browser how to display them. In simple words it uses special tags to define structure and meaning of the content.
  • Language: a set of rules to write a web page or communicate with the Browser.
Important: HTML is not a programming language because it is a markup language (it can provide structure and meaning but can't provide functionality).

Importance & Role of HTML

  • Structure: Defines how text, images, links, and media are placed.
  • Accessibility: Helps screen readers and assistive technologies understand content.
  • SEO: Search engines read HTML to rank websites.
  • Presentation: Provides meaningful structure before CSS styles it.
  • Universal: Works across all devices and browsers.

Boilerplate

A boilerplate is a basic template or starting code that contains the essential structure needed for a project.

In HTML, a boilerplate is the minimum HTML code you need to make a web page work properly in browsers.

<!-- HTML5 Boilerplate --> <!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8" /> <meta name="viewport" content="width=device-width, initial-scale=1.0" /> <title>My Webpage</title> </head> <body> <h1>Hello World!</h1> </body> </html>

Boilerplate Breakdown:

  • <!DOCTYPE html> → tells the browser this is an HTML5 document.
  • <html lang="en"> → root element and lang="en" means that the language here I am using is English.
  • <head> → meta info (title, encoding, character set, style etc.)
  • <meta charset="UTF-8"> → Supports all characters (English, Hindi, emojis, etc).
  • <meta name="viewport" content="width=device-width, initial-scale=1.0"> → Makes site responsive on mobile.
  • content="width=device-width, initial-scale=1.0"
    • width=device-width → The page width should match the device's screen width. Without this, mobile browsers might show your page zoomed out.
    • initial-scale=1.0 → The page should start with normal zoom (1x).
  • <title> → Title shown on browser tab.
  • <body> → where the visible content goes on the web site.

Some More Meta Tags

1. SEO Related

<meta name="description" content="Short description of your page"> <meta name="keywords" content="html, css, javascript"> <!-- Deprecated --> <meta name="robots" content="index, follow"> <meta name="author" content="John Doe">
  • description → summary for search engines.
  • keywords → old, ignored by Google.
  • robots → controls indexing (noindex, nofollow).
  • author → declares the page's author.

2. Social Media Sharing

Open Graph (Facebook, LinkedIn, etc.):

<meta property="og:title" content="Page Title"> <meta property="og:description" content="Page description here"> <meta property="og:image" content="image.jpg"> <meta property="og:url" content="https://example.com"> <meta property="og:type" content="website">

Twitter Cards:

<meta name="twitter:card" content="summary_large_image"> <meta name="twitter:title" content="Page Title"> <meta name="twitter:description" content="Short description"> <meta name="twitter:image" content="image.jpg">

3. Browser Control & Redirect

<meta http-equiv="X-UA-Compatible" content="IE=edge"> <meta http-equiv="refresh" content="10; url=https://example.com"> <meta http-equiv="refresh" content="30"> <!-- auto reload --> <meta http-equiv="Content-Security-Policy" content="default-src 'self'">
  • X-UA-Compatible → IE rendering engine (legacy).
  • refresh + URL → redirect after delay.
  • refresh without URL → auto reload same page.
  • CSP → adds security restrictions (e.g., no external scripts).

4. Mobile & UI Enhancement

<meta name="theme-color" content="#0d6efd">
  • theme-color → changes browser top bar color on mobile (useful for PWAs).

HTML Tags

  • HTML tags are the building blocks of an HTML document.
  • They are keywords surrounded by angle brackets < >.
  • They tell the browser what type of content it is and how to display it.

Structure of a Tag:

  • Opening tag → <tagname>
  • Content → text, image, or other elements
  • Closing tag → </tagname> (not required for self-closing tags)
  • Self closing tag → <tagname/> there is no closing tag needed and always ends with a (/).

Self-Closing Tags

Tag Description Example
<br> Line break (new line) Hello<br>World → breaks line between Hello and World
<hr> Horizontal rule / line <hr> → adds a horizontal line
<img> Image <img src="image.jpg" alt="My Image">
<input> Input field <input type="text" placeholder="Enter name">
<link> Link to external resource (CSS, favicon) <link rel="stylesheet" href="style.css">
<meta> Metadata for document (charset, viewport) <meta charset="UTF-8">
